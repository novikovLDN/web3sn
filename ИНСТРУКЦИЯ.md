# Инструкция по редактированию сайта NOVIKOV.

Здесь простыми словами — как менять контент без знания кода. Почти всё правится
в **одном-двух файлах**. После правок нужно один раз пересобрать сайт (см. в конце).

---

## 1. Карточки проектов (секция «Проекты»)

Все карточки живут в файле:

```
src/data/projects.ts
```

Внутри — список `PROJECTS`. Каждая карточка это блок в фигурных скобках `{ ... }`.
Можно **менять текст**, **добавлять** и **удалять** карточки (просто копируйте блок).

### Поля карточки

| Поле     | Что это | Пример |
|----------|---------|--------|
| `number` | Номер слева | `'01'` |
| `tag`    | Метка-статус справа | `'Кейс'`, `'В работе'` |
| `title`  | Заголовок. Перенос строки — `\n` | `'Мой\nпроект'` |
| `text`   | Короткое описание | `'Брендинг и 3D для стартапа.'` |
| `theme`  | Цвет карточки: `'orange'`, `'cream'` или `'dark'` | `'dark'` |
| `icon`   | Анимированная иконка (если нет скриншота): `'spark'`, `'cube'`, `'orbit'`, `'chip'`, `'gyro'` | `'orbit'` |
| `image`  | Скриншот проекта (необязательно) | `'/projects/my-case.jpg'` |
| `link`   | Ссылка «Смотреть проект» (необязательно) | `'https://site.com'` |

> Если указан `image` — на карточке будет скриншот. Если нет — красивая
> анимированная иконка (`icon`).

### Как добавить свой проект со скриншотом

1. Положите картинку в папку `public/projects/` (например `case-1.jpg`).
2. В нужной карточке в `src/data/projects.ts` впишите путь и, при желании, ссылку:

```js
{
  number: '01',
  tag: 'Кейс',
  title: 'Solaris\nBranding',
  text: 'Айдентика и 3D-визуал для финтех-бренда.',
  theme: 'dark',
  image: '/projects/case-1.jpg',      // ← ваш скриншот
  link: 'https://solaris.example',    // ← необязательно
},
```

### Требования к скриншотам (для максимального качества)

- **Формат:** `.jpg` (фото/рендеры) или `.webp` (легче), `.png` (если нужна прозрачность).
- **Размер:** ширина **1600–2000 px**. Не грузите гигантские файлы (>1.5 МБ) — будет тормозить.
- **Пропорции:** лучше горизонтальные (примерно 4:3 или 16:10) — так они красиво лягут в рамку.
- **Имя файла:** латиницей, без пробелов: `solaris-brand.jpg`, а не `Мой проект.jpg`.

## 2. Лента-плитки (между первым экраном и «Обо мне»)

Две строки плиток, едущие навстречу при прокрутке. Пока это **заглушки** («Скоро»
+ иконка + слово). Правятся в файле `src/sections/MarqueeSection.tsx` — список `TILES`:

```js
const TILES = [
  { word: '3D-моделинг', icon: 'cube', theme: 'dark' },
  { word: 'Рендер',      icon: 'gyro', theme: 'orange' },
  // ...
]
```
- `word` — надпись, `icon` — `spark|cube|orbit|chip|gyro`, `theme` — `orange|cream|dark`.
- Первые 6 плиток идут в верхнюю строку, остальные — в нижнюю.

Бегущая строка-текст в секции контактов — в `src/sections/ContactSection.tsx`
(константа `MARQUEE_TEXT`).

## 3. Аудиодорожка (мини-плеер справа вверху)

1. Положите аудиофайл в папку `public/audio/`.
2. Откройте `src/components/MiniPlayer.tsx` и в самом верху поправьте строку:

```js
const AUDIO_SRC = '/audio/имя-вашего-файла.mp3'
```

Поддерживаются `.mp3`, `.ogg`, `.wav`. Плеер сам запустит трек после загрузки сайта
(а если браузер заблокирует автозапуск — трек включится при первом клике/скролле).

## 4. Имя и подпись на первом экране

Файл `src/sections/HeroSection.tsx`:
- Имя — строка `NOVIKOV` (точка в конце — оранжевая, это отдельный `<span>`).
- Подпись под ним — `3D-креатор · Motion · Брендинг`.
- Нижняя фраза — `Создаю яркие и запоминающиеся 3D-проекты`.

---

## 5. Контакты и почта

Файл `src/sections/ContactSection.tsx`:
- Почта берётся из переменной окружения `VITE_CONTACT_EMAIL`, по умолчанию указана в коде.
- Соцсети — список `SOCIALS` (ссылки GitHub/LinkedIn/Instagram).

---

## 6. Как применить изменения (пересборка)

В терминале, находясь в папке проекта:

```bash
npm install      # только один раз, при первой установке
npm run build    # собрать сайт
```

Если деплой на **Railway** — просто запушьте изменения в GitHub, сборка запустится сама:

```bash
git add -A
git commit -m "обновил карточки проектов"
git push
```

Проверить локально перед деплоем:

```bash
npm run preview  # откроет локальный адрес с готовым сайтом
```

> Если команда `npm` не находится — на этом компьютере Node лежит отдельно, выполните
> сначала: `export PATH="$HOME/.local/node/bin:$PATH"`
