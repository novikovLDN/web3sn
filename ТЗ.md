# ТЗ проекта — портфолио NOVIKOV

Документ для новой сессии. Здесь всё, что нужно знать до того, как трогать код:
что делаем, в каком стиле, по каким правилам, на какие грабли уже наступили.

Читать целиком перед первой правкой. Это дешевле, чем повторять ошибки,
которые уже стоили этому проекту нескольких переделок.

---

## 1. Что это и для кого

Персональный сайт-портфолио **senior/lead дизайнера-разработчика**.

Задача сайта — не «показать работы», а **сообщить уровень и цену**. Целевой
посетитель: компания с бюджетом, которая выбирает между агентством и сильным
одиночкой. Сайт должен отсеивать тех, кому дорого, и притягивать тех, кому
нужно именно это качество.

Ключевые слова заказчика: **дорого, недоступно, технологично, погружение,
эмоция, трендово**.

**Позиционирование — лид, а не исполнитель.** Не «делаю сайты», а «проектирую
и собираю продукты, веду команды, довожу до продакшена сам».

## 2. Стек и окружение

| | |
|---|---|
| Сборка | Vite 5 |
| Фреймворк | React 18 + TypeScript 5 |
| Стили | Tailwind 3 + CSS-переменные (токены) |
| Анимация | framer-motion 12 |
| Скролл | Lenis 1.3 |
| 3D | three.js 0.169 + react-three-fiber + drei + rapier |
| Деплой | Railway, автосборка на push в `main` |

### Критично про окружение

**Node не в системном PATH.** Перед любой командой:
```bash
export PATH="$HOME/.local/node/bin:$PATH"
```

**Сборка = `tsc && vite build`.** Любая неиспользуемая переменная валит деплой.
Проверять всегда обе команды:
```bash
./node_modules/.bin/tsc --noEmit && ./node_modules/.bin/vite build
```

**Playwright намеренно НЕ в `package.json`.** Его postinstall тянет браузер и
ломает сборку на Railway. Живёт только в локальном `node_modules`. Запуск
Chromium для проверок:
```bash
--use-gl=angle --use-angle=swiftshader --enable-unsafe-swiftshader
```

**Git.** Работаем на `main`, пушим сразу после проверенной порции изменений,
не спрашивая. Проверять текущую ветку: `git push origin main` с фича-ветки
пушит локальный `main`, а не текущую работу — команда отработает успешно и
не отправит ничего. Использовать `git push origin HEAD` и сверять
`git status -sb`.

## 3. Дизайн-система — единственный источник правды

```
src/design/tokens.css     цвет, типографика, ритм, радиусы, кривые, слои
src/design/motion.ts      кривые, длительности, stagger, пружины
src/design/primitives.tsx Reveal, SplitText, Magnetic, Parallax, Counter,
                          useOffscreenPause
src/design/intro.tsx      флаг готовности после экрана загрузки
src/data/content.ts       весь копирайт + описание тона голоса
src/data/projects.ts      кейсы
```

**Правило:** в компонентах не должно появляться сырых цветов (`#ef4a23`),
произвольных отступов и длительностей. Только `var(--a)`, `var(--s-8)`,
`duration.slow` и т.п. Иначе система рассыпается и сайт снова выглядит как
набор случайных решений.

**Основная кривая появления:** `cubic-bezier(0.16, 1, 0.3, 1)` — студийный
стандарт, подтверждён независимым исследованием.

**Исключение:** тематические экраны услуг имеют **собственную палитру и
гарнитуру**, заданные локальными CSS-переменными в корне экрана. Это
сознательно — см. раздел 5.

## 4. Структура сайта

### Главная (`src/App.tsx`)

```
Hero          удар      первый экран, имя, заявление об отборе
Projects      удар      единственная световая инверсия страницы
DataStrip     переход   полоса данных: дата, координаты, живое время
Услуги        плотно    пять дверей в тематические экраны
Профиль       средне    о себе — низко и коротко
Процесс       средне    пять этапов со сроками
Хроника       пульс     датированный список выпущенного
Вопросы       плотно    FAQ, отвечает про цену прямо
Контакт       кульминация  полный вьюпорт, огромная почта, бюджет
```

**Почему такой порядок.** Это не воронка лендинга, а последовательность
премиального сайта: из 8 изученных сайтов студий **8 ведут работой**, а
разговор о себе стоит низко. Портфолио на второй позиции читается как условие
доступа; на шестой, после рассказов о себе, — как оправдание за отсутствие
работ.

**Удалены намеренно:** Marquee (плитки-заглушки), Stats (счётчики достижений
требуют доверия, которого у нового посетителя нет), Testimonials (реальных
отзывов нет).

### Тематические экраны (`src/screens/`)

| Экран | `data-screen` | Ниша |
|---|---|---|
| Веб-дизайн | `webdesign` | светлая швейцарская бумага + ультрамарин |
| Брендинг | `branding` | средний серый — как студийная циклорама |
| Motion | `motion` | глубокая петроль — затемнённая аппаратная |
| Разработка | `development` | терминальный зелёный на почти-чёрном |
| 3D-моделинг | `modeling` | умбра затемнённого павильона |
| Главная | — | тёмная тёплая |

Открываются кликом по строке услуги в `#price`. **`data-screen` на корневом
`<main>` обязателен** — по нему работают скрипты проверки, и он переживает
любой редизайн текста.

## 5. Формула экрана — эталон, одобренный заказчиком

Заказчик прямо назвал эталоном `BrandingScreen.tsx` и `WebDesignScreen.tsx`.
**Любой новый или переделываемый экран строится по этой формуле:**

1. **Тезис вместо описания.** В шапке файла — комментарий с обоснованием
   концепции и визуальной ниши. Экран доказывает компетенцию, а не
   рассказывает о ней.
2. **Разделы-законы.** Заголовки-утверждения, а не назывные:
   «Знак строится, а не рисуется», «Полигон — это бюджет, а не оценка
   качества». Не «Наши услуги».
3. **Интерактив как доказательство.** В каждом разделе — инструмент, который
   можно потрогать и **перепроверить**: сетка по клавише `G`, живая
   типошкала, WCAG-контраст, считаемый на странице, отклонение хорды по
   формуле, измеритель кадра на машине посетителя.
4. **Персистентное состояние экрана.** Один переключатель, меняющий весь
   экран разом: палитра в Брендинге, стадия пайплайна в 3D, кривая в Motion,
   условия приёмки в Разработке. Это то, что заказчик называет
   технологичностью.
5. **Своя палитра и гарнитура** локальными CSS-переменными.
6. **Блок «что остаётся у заказчика»** + процесс + закрывающее утверждение.

**Главный принцип, выведенный из обратной связи:** дорогой сайт может быть
насыщенным, но **не может быть громким**. «Множество элементов» реализуется
как плотность *реакции*, а не плотность *объектов*: один элемент, отвечающий
на десять состояний, читается технологично; десять статичных элементов
читаются шаблоном.

## 6. Тон голоса

Описан подробно в шапке `src/data/content.ts`. Коротко:

- Сдержанная авторитетность. Дорогой специалист не убеждает — он излагает.
- Конкретика вместо превосходных степеней.
- **Ни одного восклицательного знака.**
- Никаких «мы» от лица одного человека.
- Цифры только те, которые можно подтвердить.

**Правило проверки фразы:** если её можно поставить на сайт конкурента без
изменений — она не годится. Этим правилом уже был пойман собственный текст
проекта: «быстро, дёшево и хорошо — выбор из двух» оказался железным
треугольником проектного управления, то есть отраслевой банальностью.

## 7. Честность контента — жёсткое ограничение

**Ничего не выдумывать про человека.** Ни клиентов, ни наград, ни кейсов, ни
цифр результата, ни отзывов, ни дат.

Механика уже заложена в код:

| Данные | Поведение |
|---|---|
| `PROJECTS` | `filled: false` → кейс не показывается; если нет ни одного, секция показывает достойное «портфолио по запросу» |
| `TESTIMONIALS` | удалены — реальных нет |
| `CHRONICLE` | заглушки видны только в дев-режиме, в прод не попадают; секция скрыта целиком пока нет настоящих записей |
| `DataStrip` | все данные обязаны быть настоящими; выдуманное число хуже отсутствия — оно превращает единственный непроверяемый элемент в проверяемую ложь |

Фальшивая рекомендация проверяется за минуту и стоит дороже, чем отсутствие
отзывов вообще.

## 8. Грабли проекта — все ломались вживую

### Типографика на кириллице

- **Интерлиньяж ниже 0.88 срезает «й» и «ё»** строкой выше. Нельзя настраивать
  типографику на латинском «NOVIKOV» и применять к русским заголовкам.
- **Премиальные гарнитуры из англоязычных подборок часто не имеют кириллицы.**
  Уже пойманы: Instrument Serif (нет вообще), Bricolage Grotesque (нет вообще).
  Оба молча деградировали в подстановочный шрифт. **Проверять по метаданным
  Google Fonts перед подключением.**
- Разрядка прописных: кириллические шире латинских, `0.14em` вместо `0.18em`.

### Вёрстка

- **`overflow-hidden` на предке ломает `position: sticky`.** Ловилось дважды,
  в FaqSection липкость была написана, но не работала.
- **`transform: scale` не уменьшает бокс в потоке.** Рамка 1200px после
  `scale(0.3)` выглядит на 360, а места занимает 1200 → горизонтальное
  переполнение. Нужен `overflow: hidden` на обёртке.
- **Ширина анимируемого full-bleed marquee — под ~12000px.** Дальше
  превышается max texture size GPU и слой пропадает на реальных машинах.
- **JSX-комментарий `{/* */}` внутри открывающего тега — синтаксическая
  ошибка.** Ставить перед элементом. Допускалось дважды.

### Производительность — измеренные причины падения fps

- **`rAF`-циклы, работающие вне кадра.** Canvas-сетка героя перерисовывала
  ~600 окружностей каждый кадр всё время, пока открыта страница. Гейтить по
  `IntersectionObserver`.
- **`getBoundingClientRect` в кадре** — принудительный reflow. Кэшировать
  геометрию, пересчитывать по resize.
- **Бесконечные CSS-анимации не останавливаются за кадром.** Использовать
  `useOffscreenPause` из `design/primitives` (ставит `data-idle`, правило в
  `tokens.css`).
- **r3f рендерит постоянно.** `frameloop="demand"` либо монтировать сцену
  только в кадре и размонтировать при уходе.
- **Анимировать только `transform` и `opacity`.** Всё остальное вызывает
  layout или paint.

Контрольный замер: с анимациями медиана кадра 25 мс против 8.4 мс в
`reduced-motion` — так были найдены все четыре источника выше.

### Порядок появления

- **Экран загрузки — оверлей, но всё под ним уже смонтировано и анимируется.**
  Решено флагом `design/intro.tsx`; флаг поднимается по `onExitComplete`
  прелоадера, а не по `isLoading` — иначе анимация проходит сквозь уходящую
  шторку.
- Условный вызов хука (`inFrame && useHook()`) ломает React. Вызывать
  безусловно.

## 9. Как проверять

**Не верить отчётам — смотреть самому.** Скриншоты поймали баги, которые
прошли тайпчек: схлопнутый пробел («Обо мне» → «Обомне»), срезанную «й»,
заголовок FAQ, наезжающий на список, слово, разорванное посреди букв
(«3D / МОД / ЕЛИН / Г»).

Обязательный минимум перед пушем:

1. `tsc --noEmit && vite build` — обе проходят.
2. Главная и все пять экранов на **1400 и 390 px**.
3. Горизонтальное переполнение: `scrollWidth > clientWidth` — должно быть
   `false` на всех позициях прокрутки.
4. Ошибки в консоли — ноль.
5. Плейсхолдеры (`Заменить:`, `ЗАПОЛНИТЬ`, `lorem`) в прод-сборке — ноль.
6. Мёртвые ссылки `href="#"` — ноль.

Скрипт полного прогона: `scripts/shots.mjs`.

## 11. Что не сделано и почему

### Блокеры вне контроля разработчика

- **Лицензия MTS Wide.** Весь сайт набран корпоративной гарнитурой МТС.
  Использование в коммерческом портфолио вне бренда МТС требует юридической
  проверки. Если лицензии нет — меняется вся типографика сайта.
- **Портфолио пустое.** Все кейсы `filled: false`. Оболочка готова принять
  настоящие работы, но наполнить может только владелец. Это главное
  ограничение сайта, и оно не техническое.
- **Хроника — заглушки.** Заполняется владельцем, инструкция в шапке
  `CHRONICLE` в `content.ts`.
- **Цифры в текстах** — оценки, требуют замены на реальные.

### Техническое

- **120 fps не подтверждены на реальном железе.** Четыре источника лишней
  работы в кадре найдены и починены по измерениям (p95 кадра 41.7 → 25.5 мс),
  но абсолютный fps проверяем только на настоящем GPU через
  DevTools → Performance. Headless рисует программным растеризатором.
